현재 가로모드에서 식단·운동 탭은 640px 단일 컬럼입니다. 홈·통계처럼 2컬럼으로 나누는 안 — 기본 구도는 좌측 = 인사이트 위젯 / 우측 = 입력 + 오늘 기록이며, 좌우 반전 대안(A-2)도 함께 보여드립니다. 프레임은 아이폰 가로(844×390) 기준, 세로 UI는 변경 없음.

A-1식단 탭 — 좌: 위젯 / 우: 입력+기록 (추천)

추천 & 구현 메모

추천: A-1 (좌 위젯 | 우 입력+기록)

세로 모드의 배치 순서(위젯 먼저)와 일치해 두 방향의 멘탈 모델이 같고, 홈(좌 요약)·통계(좌 배너)와 "왼쪽 = 보기" 문법이 통일됩니다. 입력 중 위젯(다음 끼니·구성비)이 계속 보이므로 "얼마나 더 먹을지" 판단하며 입력할 수 있습니다. A-2(입력 왼쪽)는 입력 동선은 짧지만 방향 간 순서 불일치가 생깁니다.

구현 메모 — 두 탭 모두 세로 순서가 [위젯 N개 → 입력·기록]의 연속 분할이라, 이미 검증된 "트리 고정(DOM=세로 순서) + 컨테이너만 grid 스타일 분기" 패턴을 그대로 적용 (회전 시 검색어·입력 상태 보존). 계산·판정 로직 무변경, 세로 UI 불변, 기존 640px 캡은 960 그리드로 대체.
