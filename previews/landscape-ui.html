A안 · 홈 탭 — 세로에서 두 화면 걸치던 요약 카드가 한눈에

  • 장점 — 구현·유지비 최소(레이아웃 셸 분기만). 모든 탭·설정 모달에 일괄 효과. 화면 내부 코드는 그대로라 회귀 위험이 가장 낮음.
  • 한계 — 카드가 640px로 넓어질 뿐 배치는 세로와 동일. 좌우 공간을 "더" 쓰지는 못함.

B안A + 카드 2컬럼 그리드 — 정보 밀도 극대화

A안 셸 위에, 세로로 쌓이던 카드를 가로에서 2컬럼으로 재배치. 요약과 오늘의 기록(식단·운동)이 스크롤 없이 한 화면에 들어옵니다. 카드 폭은 세로 모드(~440px)와 비슷한 ~350px라 내부 레이아웃이 자연스럽게 재사용됩니다.

B안 · 홈 탭 — 좌: 요약 / 우: 식단·운동 기록

  • 장점 — 가로 화면다운 정보 밀도. 기록 확인(오른쪽)과 달성 현황(왼쪽)을 동시에 봄. 식단·운동·통계 탭에도 같은 원리 적용 가능.
  • 비용 — 탭마다 "가로일 때 어떤 카드를 어느 컬럼에 둘지" 배치 설계가 필요. 작업량은 A의 3~4배, 이후 새 카드 추가 때마다 배치 고려 필요.

C안가로 회전 = 차트 확대 모드 — 체성분·통계 특화

금융 앱 패턴: 체성분·통계 탭에서 가로로 돌리면 차트가 화면 전체로 확대됩니다. 1개월 시계열의 X축이 세로 모드 대비 약 3배 넓어져 추세·변곡이 훨씬 잘 보입니다. 다른 탭은 현재처럼 두거나 A안 셸과 병행.

C안 · 체성분 탭 가로 회전 — 차트 전용 캔버스

  • 장점 — 긴 시계열(3개월·전체) 판독에 압도적. 구현 범위가 차트 카드 하나로 좁아 작업량은 A와 비슷.
  • 한계 — 체성분·통계 외 탭엔 이득 없음. 의도치 않은 회전에도 화면이 확 바뀌는 "놀람"이 있어 발견성·예측성은 낮음. A안과 병행 가능(가로 기본 = A 셸, 차트 카드에서만 확대).

추천: A안으로 시작 → 써보고 B로 확장

A는 코드 몇 곳의 조건 분기로 끝나 회귀 위험이 가장 낮고, 모든 화면이 즉시 좋아집니다. 실사용 후 "가로에서 더 많이 보고 싶다"면 B를 홈 탭부터 점진 적용하는 걸 권합니다. C는 A와 배타적이지 않아서, 차트를 자주 보신다면 A 위에 얹는 옵션으로 나중에 추가할 수 있습니다.

구현 메모 — matchMedia('(orientation: landscape)') 기반 useOrientation 훅 1개 + 조건부 인라인 스타일로 구현. 계산 로직(utils.js)·판정·데이터는 일절 건드리지 않는 순수 UI 분기입니다. 세로 UI는 어떤 안에서도 변경 없음. 렌더 스모크 테스트는 세로(기본) 기준 유지, 가로 분기는 훅 mock으로 커버.
